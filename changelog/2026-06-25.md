# 2026-06-25 改动总结：站立任务改造为速度跟踪行走 + sim-to-real 噪声鲁棒化

## 背景

真机测试发现两个问题：
1. **站立时两腿快速交替抖动** —— 根因是训练时观测**完全没有噪声、没有动作延迟**，策略学成了对干净观测高度敏感的高增益控制器，上真机遇到 IMU/电机噪声和通信延迟就抖。用户采的 30s 静置 IMU 数据（`imu_noise_static_2026-06-25.csv`）显示噪声其实很小（gyro std ~0.003 rad/s），证实静置噪声不是主因，真正缺的是 domain randomization。
2. **频繁小碎步、迈不开** —— 根因是当前任务**根本不是行走**：`_commands` 恒为 0，`lin_vel` 奖励实际在惩罚一切机体移动，这是个"零指令站立 + 抗推力"任务。reward 里也没有任何步态/接触项，策略对"怎么迈步"无偏好，收敛到原地碎步这个局部最优。

用户确认最终目标是**速度跟踪行走**，命令维度为**仅 vx 前进**，并**保留原地站立能力**（部分 env 采样零速度指令）。

涉及文件：
- `source/oceanisaaclab/oceanisaaclab/tasks/direct/oceanisaaclab/oceanisaaclab_env_cfg.py`
- `source/oceanisaaclab/oceanisaaclab/tasks/direct/oceanisaaclab/oceanisaaclab_env.py`

## 改动 1：sim-to-real 噪声鲁棒化（解决抖动）

### 观测高斯噪声
cfg 新增 `enable_obs_noise` + 4 个 std；`_get_observations` 在原始物理量上加噪后再乘 scale。

| 参数 | 值 | 说明 |
|------|-----|------|
| `noise_ang_vel` | 0.03 rad/s | 静置实测 ~0.003，留余量覆盖运动振动 |
| `noise_proj_g` | 0.02 | 静置实测 ~0.0004，覆盖姿态估计误差 |
| `noise_joint_pos` | 0.01 rad | 编码器噪声 |
| `noise_joint_vel` | 1.0 rad/s | **差分速度噪声大，常为腿抖主因** |

### 动作延迟随机化
cfg 新增 `enable_action_latency` / `action_latency_steps=2`。`__init__` 加 action 环形缓冲 + 每 env 随机延迟；`_pre_physics_step` 把延迟后的动作应用到物理端（观测仍喂策略自己刚下发的 `_actions`，符合真机语义）；`_reset_idx` 重置缓冲并重采样延迟步数。

## 改动 2：行走任务改造（解决碎步）

**关键：观测维度不变（仍 39）**，`commands(3)` 早已在观测里，只是从没非零采样过，无需改网络结构。

### 命令采样（`_reset_idx`）
- 按 `command_vx_range=(-0.3, 0.8)` m/s 采样 vx（前进为主，允许少量后退）；vy、yaw 保持 0。
- `stand_still_prob=0.2`：该比例 env 置零速度指令，保留站立能力；用 `_is_standing` 布尔 mask 记录，门控 air-time 奖励。

### 接触传感器（cfg + `_setup_scene`）
- cfg 新增 `ContactSensorCfg`，监测双脚 `leg_[lr]5_link`，`track_air_time=True`。
- **踩坑修复**：Ocean URDF 把腿 link 导入成**嵌套**结构，IsaacLab 的 `activate_contact_sensors` BFS 到 `base_link` 就停，脚端拿不到接触 API。在 `_setup_scene` 里 clone 之前手动给脚端 prim 补 `PhysxContactReportAPI`（`_activate_feet_contact_sensors`）。
- 接触传感器索引（`_feet_contact_ids`）与 articulation 索引（`_feet_body_ids`）是两套独立索引，均用 `preserve_order=True` 按 [r5, l5] 顺序解析对齐。

### 奖励重构（`compute_rewards`）
| 奖励项 | 改动 | 说明 |
|--------|------|------|
| `track_lin_vel` | **新增** | exp 跟踪 vx 指令，正号，行走核心驱动 |
| `lateral` | **新增** | 惩罚非指令的 vy 侧移和 yaw 自转 |
| `ang_vel` | **改** | 从罚全三轴改为只罚 roll/pitch（避免与转向冲突） |
| `feet_air_time` | **新增** | 落地瞬间奖励腾空时长，仅非站立 env，碎步克星 |
| `feet_slide` | **新增** | 接地时惩罚水平滑移 |
| `stand_still` | **新增** | 站立 env 惩罚脚移动 |
| `action_rate` | **改** | 惩罚从 -0.01 加大到 -0.02 抑制高频抖动 |
| 保留 | — | alive / terminated / upright / height / joint_pos / joint_vel |

## 验证

- 两文件 `py_compile` 通过。
- 64 env × 3 iteration 短训冒烟跑通，13 个奖励项全部正常更新（`track_lin_vel≈4.07` 等），无 shape/runtime 错误。
- 专项验证：接触传感器精确解析到 2 只脚，两套索引正确对齐，所有张量 shape 正确，命令/站立 mask 采样正常。

## 首轮训练观察（rsl_rl/bdx_stand_direct/2026-06-25_11-55-05，~1700 步）

机制正常但有警示信号：
- ✅ `mean_reward` 稳步升到 ~1130 仍在涨；`time_out` 从 0 升到 ~8，存活变长。
- ⚠️ **`feet_air_time` / `feet_slide` / `stand_still` 三项持续变负且越来越陡**，与 ~1400 步的 reward 跃升同步 —— 策略靠"碎步 + 脚打滑"的低质量步态涨分，**步态惩罚权重太弱压不住**。
- 量级失衡：`upright≈7.7`、`height≈5.4`、`track_lin_vel≈4.1`，而 `feet_air_time` 仅 -0.05~-0.46，差 1~2 个数量级。
