# 2026-06-30 低速行走第一阶段奖励（开源配方对齐，保留零速站立）

文件：`oceanisaaclab_env.py` / `oceanisaaclab_env_cfg.py`

## 背景
- 06-26 的策略可以稳定站立，但侧推时迈步不理想：能靠踝/髋顶住就不迈，顶不住才急摆腿。
- 根因已不只是推力课程或 `air_time_target`，而是任务仍接近纯站立：`stand_still_prob=1.0`，且 `stand_still` 在稳定状态下总是惩罚脚动。正常行走时只要没失衡，脚动也会被罚，策略自然学成钉地/贴地小碎步。

## 参考的开源配方
- `legged_gym`：`feet_air_time` 只在非零移动命令时给，零命令时不鼓励抬脚；`stand_still` 只在速度命令接近 0 时生效。
- Booster T1 humanoid 配方：命令里保留 `still_proportion`，同时使用速度跟踪、`feet_slip`、`feet_swing`、足间距/姿态等项，避免只靠站立稳定项学行走。

## 修改思路
- 不直接上强相位步态（需要把 gait phase 加进观测，会破坏当前 39 维策略接口，无法平滑从 `model_4000` 续训）。
- 采用第一阶段“弱步态结构”：保留现有观测维度，打开低速前进命令，用奖励把“正常离地迈步”从“贴地搓步”里分出来。

## 具体改动
- `stand_still_prob`: `1.0` → `0.35`。约 35% env 训练零速站立，其余训练低速前进。
- `command_vx_range`: `(-0.3, 0.8)` → `(0.15, 0.45)`。第一阶段只做低速前进，先让步态成形。
- 新增 `move_command_threshold=0.10`：低于阈值按站立命令处理。
- `stand_still` 改为仅在“零速命令 + 稳定”时惩罚脚动；非零速度命令和失衡恢复时放行脚步。
- `feet_air_time` 改为在“非零速度命令或失衡”时给；`air_time_target=0.18`，短碎步落地仍会扣分。
- `feet_slide` 在行走时保持启用，逼策略靠抬脚摆动而不是脚贴地滑动。
- 新增：
  - `single_support`：非零速度命令下奖励单脚支撑，打破双脚同时搓地。
  - `no_fly`：惩罚双脚同时离地，避免跳步。
  - `feet_clearance`：摆动脚低于 `foot_clearance_target=0.035m` 时惩罚，防止贴地拖脚。
- 行走/失衡时放松 `joint_vel`、`action_rate`、`contact_force_rate` 权重，零速稳定站立时保持严格抗抖。

## 建议训练
继续从 06-26 的稳定甜点 `model_4000` 续训，不要从后期过度钉地/高 fall 的 checkpoint 续：

```bash
./_isaaclab/isaaclab.sh -p scripts/rsl_rl/train.py \
  --task Ocean-BDX-Stand-Direct-v0 \
  --num_envs 4096 --max_iterations 5000 \
  --resume --load_run 2026-06-26_10-11-23 --checkpoint model_4000.pt
```

## 观察指标
- `track_lin_vel` 应上升，`feet_air_time` 不应长期为负。
- `single_support` 应明显为正，表示出现交替支撑。
- `feet_slide` 不应暴涨；若速度跟上但 `feet_slide` 很大，说明还在搓地。
- 零速样本的 `stand_still`、`contact_force_rate` 不能明显退化。
- 若 1000-2000 iter 后仍贴地小碎步：提高 `foot_clearance_target` 到 `0.05` 或把 `rew_scale_feet_clearance` 绝对值加大。
- 若会迈步但摔：先把 `command_vx_range` 缩到 `(0.10, 0.30)`，或把 `stand_still_prob` 提到 `0.5` 稳住课程。

## 训练后复盘：不是 5000 iter 太少，是任务定义仍缺转向/相位结构

`2026-06-30_10-35-37` 训到 8999 后，sim2sim 切 RL 抖动，按上下左右/旋转仍不抬腿。TensorBoard 现象：
- `feet_air_time` 长期为负，`feet_slide` 继续变差，说明没有学出离地摆腿，而是在接触面上抖/搓。
- `entropy` 一路升高，动作分布没有收敛到稳定步态。
- `stand_still` 仍有明显惩罚，说明策略还在零速稳定和行走目标之间拉扯。

根因：
- 实际 run 是从 `model_4000` 续训，`agent.yaml` 里 `resume: true`；旧站立策略和 obs normalizer 都偏向零命令站立。
- 环境只采样/奖励 `vx`，没有采样/奖励 `wz`。sim2sim 左右旋转命令对策略是训练分布外输入，奖励里还把 yaw 当作非指令角速度惩罚，当然不会抬腿转向。
- 39 维无相位观测的 MLP 在小双足上容易找到“站着抖/贴地搓”的局部最优；只靠 `single_support` 和 air-time 不足以稳定地产生交替支撑。

### v2 修改（从头训练，不兼容旧 checkpoint）
- 观测 39→41：在命令后加入 `gait_clock=[sin(phase), cos(phase)]`。
- 新增 `command_wz_range=(-0.8,0.8)`、`turn_in_place_prob=0.25`，训练原地转向和行走转向。
- 新增 `track_ang_vel` 奖励；`lateral` 不再惩罚 yaw，只惩罚非指令侧移。
- 新增 `phase_contact`：按 gait clock 奖励左右脚交替接触，打破无相位抖动/搓地局部最优。
- `foot_clearance_target` 提到 `0.05m`，`feet_clearance` 绝对值加大。
- 从头训练推力课程放柔：`push_force_range=(8,20)N` → `push_force_range_max=(35,70)N`。
- PPO 新实验名 `bdx_walk_phase`，默认 `max_iterations=15000`，`entropy_coef=0.001`。

### v2 训练命令
不要加 `--resume`，旧 checkpoint 观测维度不同，且旧 normalizer 会拖累行走学习：

```bash
./_isaaclab/isaaclab.sh -p scripts/rsl_rl/train.py \
  --task Ocean-BDX-Stand-Direct-v0 \
  --num_envs 4096 --max_iterations 15000 --headless
```

### v2 部署/验证注意
- 新策略观测是 41 维，sim2sim/实机侧必须在 `command` 后补 `sin/cos gait clock`。
- sim2sim 键盘输入需要映射到 `[vx, 0, wz]`，左右旋转用 `wz`，不要只改可视化相机或外部 base 姿态。
- 若 3000-5000 iter 后 `phase_contact` 上升、`feet_air_time` 接近 0 或转正、`feet_slide` 下降，再看 sim2sim；若这些曲线不对，继续训练通常也不会“自然变好”。
