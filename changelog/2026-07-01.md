# 2026-07-01 低速行走奖励地形重构：打破"站着不动"局部最优

文件：`oceanisaaclab_env.py` / `oceanisaaclab_env_cfg.py` / `agents/rsl_rl_ppo_cfg.py`
新增诊断：`scripts/diag_walk.py`（强制命令行走诊断）、`scripts/analyze_tb.py`（离线解析 TB 曲线）

## 背景
06-30 版加 gait phase 从头训 15000 iter，sim2sim 里 RL 抖动抬头、给 vx 命令不迈步、只有右向有迈步趋势会摔。

## 定量诊断（对 06-30 的 model_14999，命令 vx=0.3、关闭推力）
用 `diag_walk.py` 实测：
- 实际前进 vx = **-0.004 m/s**（跟踪比 -0.01）：完全不迈步、不前进。
- 每只脚接触比例 **0.995**、双脚同时着地 **0.989**，340 步只离地 0.6 次：几乎从不抬脚。
- 摆动脚只比站立高 1.8cm（脚 link 原点站立态就 0.067m）。
- roll/pitch 角速度 p95=0.77、max=6.51：抖动。

用 `analyze_tb.py` 解析曲线（每秒均值末段）：
- 正奖励：upright 87.9 + height 49.3 + alive 51.0 + track_lin_vel 86.9 ≈ **275/s（站着就能拿）**。
- 迈步激励：phase_contact 17 + single_support 2.1 + feet_air_time **-0.84** ≈ 18/s（一项还是负的）。
- `feet_clearance` 全程 **-0.0000**（恒失效）；entropy 从 -3.5 单调升到 22.6（动作分布发散）。

## 根因（逐条实测验证，不是几何问题）
零位=膝微屈承重站姿，几何上完全能迈步（迪士尼 BDX 同姿态起步）。问题是**站着不动就是奖励最优解**：
- **E 站立项主导正奖励**：站着能拿 ~94% 正奖励，迈步纯亏。
- **C track_lin_vel 惩罚太弱**：`2.0*exp(-err²/0.25)`，vx=0.3 站着不动仍拿 1.40/步（满分 70%）。
- **A feet_clearance 度量错误恒为 0**：度量的是脚 link 原点高度（站立就 0.067m > target 0.05），`clamp(0.05-0.067)=0` 永远不触发。
- **B feet_air_time 长期为负**：target=0.20 压在摆动时长上沿，短步落地倒扣 → 净惩罚迈步。
- **D phase_contact 区分度不足**：duty=0.58 双脚永远着地白拿 phase_match=0.58，软 L1 逼不出交替。
- **F entropy 发散**：是 E 的症状（奖励地形无单一最优），不单独治。

## 具体改动（纯训练侧，不碰 default 姿态/action_scale/观测维度/oceanbdx.yaml）

### cfg
- `lin_vel_track_sigma` 0.25→**0.06**：站着不动从 1.40/步 压到 ~0.36，迈步跟上仍满分。
- `rew_scale_upright` 2.0→**1.0**、`rew_scale_height` 1.0→**0.5**、`rew_scale_alive` 1.0→**0.5**：降站立正奖励。
- `rew_scale_track_lin_vel` 2.0→**2.5**：提行走主驱动。
- `stand_still_prob` 0.25→**0.15**：削弱站立样本占比。
- `gait_duty_factor` 0.58→**0.5**：明确交替，双支撑窗口最小。
- `air_time_target` 0.20→**0.15**：匹配 0.6s 周期/duty 0.5 的实际摆动时长。
- 新增 `foot_origin_offset=0.067`：脚 link 原点到脚底的结构偏移，用于校正 clearance 度量。
- `rew_scale_feet_slide` -1.0→**-0.5**：前期减弱，避免压制正常摆动落地微滑。
- 新增 `rew_scale_swing_contact=-1.5`（摆动相仍接触的硬惩罚）；`rew_scale_phase_contact` 0.8→**1.5**（支撑相纯正奖励）；`rew_scale_single_support` 0.4→**1.0**；`rew_scale_feet_clearance` -30.0→**+1.0**（改正奖励）。
- 新增命令幅度课程：`enable_command_curriculum`、`command_vx_range_start=(0.15,0.30)`、`command_curriculum_steps=72_000`（≈3000 iter×24）；`command_vx_range` 终点改 (0.10,0.35)。

### env（`compute_rewards` / `_get_rewards` / `_reset_idx`）
- `feet_height` 减去 `foot_origin_offset`，语义变为**脚底离地间隙**（修 clearance 恒 0）。
- `feet_air_time` 落地奖励 `clamp(air_time-target, min=0)`：短步不再倒扣，消除"迈短步比不迈更亏"死锁。
- phase 拆成两项：支撑相奖励接触（`support_match`）+ 摆动相硬惩罚接触（`swing_violation`）。去掉恒接触白拿的软 L1。
- `feet_clearance` 改摆动相正奖励 `exp(-(clearance-target)²/0.0025)*swing_mask*moving_command`。
- `motion_smooth_weight` 行走地板 0.35→**0.2**：行走时更放开关节。
- `_reset_idx`：对 ~50% 有 vx 命令的 env 注入初速度 `default_root_vel[:,0]≈command_vx`（reference-state init），降低从静止迈第一步的 bootstrap 难度。
- 新增 `_current_command_vx_range()`：按 `common_step_counter` 线性放开命令范围（复用 push 课程机制）。

### PPO
- `init_std` 0.4→**0.3**：减小初期乱抖。`entropy_coef` 0.001 保持（发散是奖励地形问题）。

## 奖励+算法二次调整（治"原地踏步不前移"+发散）
核心判断：步态整形奖励（phase_contact/single_support/feet_clearance/feet_air_time）与**实际前进速度解耦**，导致"原地踏步"能白拿步态奖励，成了替代旧"站着不动"的新局部最优；且 `lin_vel_track_sigma=0.06` 仍偏松，原地踏步（vx≈0）还能骗到 track_lin_vel 峰值 ~40%。

### env（`compute_rewards`）
- 新增**前进进度门控** `fwd_gate = clamp(vx_actual/vx_cmd, 0, 1)`：仅对前进命令生效（转向/站立命令 gate=1 不受影响），失衡时取 `max(fwd_gate, instability)` 解除门控保推力恢复。用它乘 `phase_contact / single_support / feet_clearance / feet_air_time` → 原地踏步这些奖励趋近 0，只有真前进才拿得到。
- 新增**线性前进奖励** `rew_progress = rew_scale_progress * fwd_frac * forward_command`：exp 跟踪项在 vx≈0 附近梯度太平，加线性项给持续"往前走"梯度；站立/原地转向为 0，不可 farm。

### cfg
- `lin_vel_track_sigma` 0.06→**0.04**；`rew_scale_track_lin_vel` 2.5→**3.0**；新增 `rew_scale_progress=1.5`。
- 步态整形 scale 下调（配合门控降总量级/方差）：`phase_contact` 1.5→**1.0**、`single_support` 1.0→**0.6**、`feet_clearance` 1.0→**0.6**、`feet_air_time` 5.0→**3.0**；`swing_contact` 保持 -1.5。
- `rew_scale_ang_vel` -0.1→**-0.06**：该项为站立调稳，行走时太强会压制自然躯干摆动。

### PPO（治 mean_std 爆炸/发散）
- 无 std 上限配置项，改从根治：`std_type="log"`（std 对数参数化，更新更平滑，抑制跑飞）+ `num_learning_epochs` 5→**4**（减少对噪声 advantage 过拟合）。降奖励量级/方差利于 value 收敛。

### 冒烟验证（64 env 随机动作 150 步）
total reward 无 NaN/Inf；`progress` mean 0.55/max 2.62 且 min=0（非前进命令确为 0）；phase_contact/single_support/feet_clearance 门控后正常；forward-command 占比 ≈0.78。

### 调参回退指引（若下轮训练又跑偏）
- 若**又不肯迈步**（门控太狠，bootstrap 起不来）：调大 `command_vx_range_start` 下限或把 `progress_pay` 门控放软（如 `0.3 + 0.7*fwd_gate`）。
- 若**迈步但仍原地/前移弱**：继续收 `lin_vel_track_sigma`（→0.03）或提 `rew_scale_progress`。
- 若**仍后仰/不稳**：优先确认前移是否真起来（diag 实测 vx），后仰多为不前移的副产物；必要时再单独加躯干 pitch 目标项。
- 若 `Policy/mean_std` 仍爆：进一步降步态 scale，或 `entropy_coef` 0.001→0。

## 第二轮调整（只动命令课程，不加姿态塑形）
讨论结论：**不加"按速度给目标前倾角"这类姿态塑形**——那是替策略把"该用什么姿态走"写死，易过拟合手工姿态、伤鲁棒性与 sim2real。后仰交给现有通用信号（upright / progress / terminated 不摔）自行解决。

只改命令课程——不是塑形，是"别逼它做还做不到的速度"，让能力自己长：
- `command_vx_range` 上限 (0.10,**0.35**)→(0.10,**0.25**)：先把能走稳的速度练扎实（实测 0.3 后仰摔），以后再抬上限。
- `command_vx_range_start` (0.15,0.30)→**(0.10,0.18)**：一开始只学慢速走稳，别一上来就要求 0.3。
- `command_curriculum_steps` 72_000→**96_000**（≈4000 iter）：放慢，每个速度档有时间巩固，避免"课程放开>能力增长"导致的过峰值退化。

预期：把"超速摔"的 regime 从训练中移除后，过峰值退化应转为持续改善，value loss 也应随摔倒方差下降而回落。验证同前（~3000 iter diag 交叉确认 + 看 fall/value 是否不再回升）。若走稳后想提速，再逐步上调 `command_vx_range` 上限。

## 第三轮调整（课程时序重构：先学走再学抗推 + 降推力 + 拉长总时长）
讨论点：①用户训练用 num_envs=8192，需澄清 `common_step_counter` **不乘 num_envs**（每控制步+1，1 iter=24 step），所以 8192 vs 4096 课程走满的 **iter 数完全一样**，8192 只是每 iter 采 2 倍数据、学得更充分。②推力课程和命令课程从 iter 0 同时爬，"还没会走就被大推力砸"是叠加干扰。③实测机身总质量 **10.0kg**，`push_duration_s=0.18s` 下 70N → Δv=1.26 m/s（走速 0.25 的 5 倍），是"踹飞"不是"逼迈步恢复"。

改动：
- **分阶段错开**：`_current_push_force_range` 加启动延迟 `push_curriculum_start_step=120_000`（≈5000 iter），此前推力保持地板 (8,20)N，命令课程走满后推力才开始爬。等于"先训走、再训抗推"，无需分两次训练。
- **降推力**：`push_force_range_max` (35,70)→**(20,40)N**（Δv 0.36~0.72，有挑战但可恢复）。
- **拉长**：`command_curriculum_steps` 96k→**120k**（5000 iter）；`push_curriculum_steps` 60k→**120k**（从 start_step 起算，iter 10000 到顶）；`max_iterations` 15000→**20000**，留 iter 10000~20000 共 1 万 iter 满难度巩固。

时间线（已用纯 python 核算）：

| iter | vx 上限 | 推力上限 | Δv(10kg) | 阶段 |
|---|---|---|---|---|
| 0 | 0.18 | 20N | 0.36 | 学走 |
| 5000 | 0.25(满) | 20N | 0.36 | 走稳 |
| 10000 | 0.25 | 40N(满) | 0.72 | 加抗推 |
| 20000 | 0.25 | 40N | 0.72 | 巩固 |

注：训练命令需 `--max_iterations 20000`（或不加该 flag 用 cfg 默认）。若走稳后想提速/加强抗推，再逐步上调 `command_vx_range` 上限与 `push_force_range_max`。前 5000 iter 若想更"纯净"地学走，可把地板 `push_force_range` 也调更小甚至 (0,0)。

### sim2sim 实测（model_19999）
- 切 RL 后踏一步多即可稳定站住；E 旋转能稳定正常迈步旋转；左右低速可迈步走。摔倒少，明显比前几版好。
- **重大发现：W/S 方向与预期相反**——按 W 机器人往「物理后方」走（原以为是前进），且**往该方向低速能正常迈步走**；按 S（真正头朝向的前进）：0.1 站姿正常不后仰，0.2 开始前倾、想抬腿又不敢，再大就前倾摔。**前进方向没学会抬腿**。
- 站立姿态仍**仰头 + 重心靠后**。
