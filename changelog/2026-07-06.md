## 0. 论文理解要点（回答「观测里没有命令？」的疑问）

- 论文策略是 π(a_t | s_t, φ_t, g_t)：s_t（式 8）里确实没有命令，但命令 g_t 是
  **独立的条件输入直接进策略网络**。行走策略的 g_peri =（头部命令, v_P, ω_P）（式 6），
  前后/横移/转向速度就是直接喂进网络的——与本工程 obs 里的 cmd3 同一回事。
- 命令**同时**走第二条通路：参考生成器 f(f_t, φ_t, g_t)（式 2-4）按同一份命令实时
  合成参考轨迹进模仿奖励（艺术家给几个速度档参考步态，按命令程序化混合 [14]），并输出
  随命令变化的相位速率 φ̇。所以是「命令既直接进策略、又调制参考动作」，两者并行。
- 蹦跳/舞蹈等离散动作**不是切换参考序列喂同一个策略**，而是**切换到独立训练的
  episodic 策略**（divide-and-conquer，图 9 展示策略切换）；摇头点头等头部动作是把
  动画混合成头部命令偏移喂给当前策略（本轮固定脖子，整体不做）。
- 论文**没有显式速度跟踪奖励**：速度跟踪由「躯干 path 系位置逐帧贴参考」隐式实现
  （path frame 按命令速度积分前进，跟不上就没位置奖励）。

## 11. 答疑并补文档：sim2sim/sim2real 中命令 (vx, vy, wz) 如何起作用

问题：「gen_reference_gait.py 生成了各命令下的不同轨迹，观测里又没有 cmd 了，
sim2sim 是不是要按 cmd 修改观测中的参考轨迹？」——**否，两个前提都不成立**：

- 观测里**仍然有 cmd**：57 维观测最后 3 维就是 `cmd × command_scale(2,2,1)`
  （`oceanisaaclab_walk_env.py` `_get_observations` 末项），没有被去掉；
- **参考轨迹从不进观测**：npz 步态库（各命令网格点的周期轨迹）只在训练时用于
  ① 表 I 模仿奖励的目标状态采样、② RSI 重置初始化。观测里只有机器人自身状态 +
  path frame 相对量 + 相位谐波 + cmd，部署侧不需要采样参考关节角。

cmd 在部署 runtime 中通过三条通道起作用（sim2sim / sim2real 程序需要实现）：

1. **obs[54:57] = cmd × (2.0, 2.0, 1.0)**。cmd 为头部系 (vx 前+, vy 左+, wz 逆时针+)，
   限幅到训练范围 vx±0.25 / vy±0.15 / wz±0.8。
2. **path frame 状态机**（逐行为复刻 `path_frame.py`，50Hz，dt=0.02s）→
   obs[0:2]=path 系躯干 xy×4.0，obs[2:4]=sin/cos(躯干 yaw − path yaw)：
   - moving = any(|cmd|) > 0.08：pos += R(yaw)·(vx,vy)·dt，yaw += wz·dt；
   - 否则：pos/yaw 一阶低通（α=min(1, dt/1.0)）收敛到双脚中心 xy / 躯干 yaw；
   - 收尾投影：‖pos−躯干xy‖ 钳到 ≤0.25m，|yaw−躯干yaw| 钳到 ≤0.6rad；
   - 依赖躯干世界位姿估计（里程计/状态估计器）与双脚中心（FK 可得）。
3. **相位积分** → obs[50:54] 谐波 sin/cos(2πφ)、sin/cos(4πφ)：
   φ̇ 从 npz `phase_rate` 表按 cmd 三线性插值（复刻 `reference_gait.py`
   `sample_phase_rate`；恒定步频库即常数 1/gait_period），φ ← (φ+φ̇·0.02) mod 1，
   零命令相位照常推进。

因此「控制前后左右」= 改 cmd 三个数：cmd 进 obs 尾部、驱动 path frame 锚点按
命令速度移动（策略贴不住锚点拿不到奖励 → 隐式速度跟踪）、调 φ̇ 改步频。
部署侧只需从 `reference_gait.npz` 读 `phase_rate` + `vx_grid/vy_grid/wz_grid`
四个数组，其余轨迹表（joint_pos/joint_vel/…）不用加载。

- 涉及文件：README.md 新增「路线 B 部署（sim2sim / sim2real）」小节（本节内容的
  规范版），oceanbdx 部署工程按上述三通道改造（本仓库外）。

## 12. 修复反射惯量写入 API 报错（首次冒烟踩到第 8 节待办）

`oceanisaaclab_walk_env.py` `_write_armature` 调用 IsaacLab 3.0.2 的
`write_joint_armature_to_sim_index()` 时用位置参数传 `armature`，但该 API 已把
`armature` 改成**关键字专用参数**（`def ...(self, *, armature, ...)`），环境创建
（`__init__` 里首次写标称反射惯量）即崩：

```
TypeError: write_joint_armature_to_sim_index() takes 1 positional argument
but 2 positional arguments (and 2 keyword-only arguments) were given
```

`except TypeError` 回退分支同样是位置参数，两条路都失败。改法：两个分支都用
`armature=armature.contiguous()` 关键字传参（旧 deprecated `write_joint_armature_to_sim`
也接受关键字，兼容）。核查了其余 sim 写入调用（`set_joint_effort_target_index`、
`write_joint_position/velocity_to_sim_index`）均已关键字传参，无同类问题。
16 env × 2 iter 冒烟通过（环境创建→步进→学习）。

## 13. 修奖励地形：站立局部最优复发（训 3000 iter 不迈步）

路线 B 换成论文表 I 隐式速度跟踪后，训 3000 iter（`logs/.../2026-07-06_12-45-07`）
仍不迈步、原地小幅扭动。解析 Episode_Reward 分解定位为站立局部最优复发
（见 memory `ocean-walk-standing-local-optimum`）：

- `survival` episode_sum ≈19.57，占总回报 181 的 **93%** → 策略只需「别摔」就锁定回报，
  迈步的摔倒风险不划算；
- `torso_pos_xy`/`torso_orient`（迈步跟踪主项）得分极低（0.36/0.25），因 exp 核
  `rew_k_torso_pos_xy=200` 太陡（位置差 0.09m exp 即塌到 ≈0.2）→ 跟踪梯度稀疏；
- `leg_joint_pos`=-0.79（负 L2）→ 策略主动远离参考迈步姿态；
- 参考库本身健康（vx=0.25 膝摆 0.54rad、踝 0.49rad、接触占空比 50%，是真迈步），
  命令采样也含行走命令 → 纯奖励地形问题。

改法（`oceanisaaclab_walk_env_cfg.py`，**最小变量只动两项**）：
- `rew_w_survival` 20.0 → **5.0**（论文原值依赖强密集模仿信号才不吞任务梯度，
  本工程信号稀疏时压倒一切；降到 5 仍抑制「主动摔」但不再主导）；
- `rew_k_torso_pos_xy` 200.0 → **60.0**（放平位置跟踪核，让 0.15m 偏差仍有 ≈0.26 梯度）。

其余（朝向核 k、`rew_w_leg_joint_pos`、`rsi_prob`）暂不动，观察效果。改后须从头重训
（奖励结构变化，旧 checkpoint 作废）。若仍不迈步，下一步再降朝向核 / joint_pos 权重 /
提 rsi_prob 或加 warmup 强制迈步。
