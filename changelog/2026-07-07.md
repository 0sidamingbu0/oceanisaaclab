# 2026-07-07 路线 B 站立局部最优第 3 次复发:奖励地形 + 探索崩塌联合修

07-06 改法(`rew_w_survival` 20→5、`rew_k_torso_pos_xy` 200→60)后训到 5800 iter
(`logs/rsl_rl/bdx_walk_imitation/2026-07-07_07-38-12`),sim2sim 仍是**站住 + 轻微
扭动 + 不抬腿 + 蹭着走**。方向对但不够。用 `scripts/analyze_tb.py` 离线解析
Episode_Reward / Policy / Loss 分解,定位到**两个近因叠加**,均属站立局部最优
(见 memory `ocean-walk-standing-local-optimum`)。

## 1. 诊断(analyze_tb 实测,run 2026-07-07_07-38-12,~5800 iter)

| 信号 | start → end | 含义 |
|---|---|---|
| `Policy/mean_std` | 0.082 → **0.013**(init 0.3,缩 23×) | **探索死了**;entropy_coef=0 无物阻止 std 崩塌 |
| `Loss/entropy` | -12.7 → **-29.1** | 策略几乎决定论化,锁死"蹭着站" |
| `Termination/fall` | 12.4 → **0.16** / `time_out` 20.5 | 学到"绝不摔、活到 timeout" = 站立最优本身 |
| `survival` | → **4.96** | 是前进项 `torso_pos_xy`(0.43)的 **11 倍**,降到 5 仍主导 |
| `torso_pos_xy` | 0.40 → **0.43**(几乎不动) | 唯一前进驱动项是平的 = 没往前走 |
| `leg_joint_pos` | -0.78 → **-0.69**(几乎不动) | 策略宁愿吃偏离迈步姿态的惩罚也要站着 |

**近因 A —— 探索崩塌:** walk PPO cfg `entropy_coef=0`(论文原值)。策略在发现"迈步比
站着值"之前 std 就塌到 0.013、entropy -29,锁死站立盆地。论文能用 entropy=0 是因其
模仿信号又密又主导;本工程各模仿项弱、盆地赢。⚠ 与 07-01 walk_phase 的 std **发散**
(changelog/2026-07-01.md)方向相反,这次是 std **崩塌**,是不同状况。

**近因 B —— torso_pos_xy 核仍太陡:** k=60 时 path anchor 按命令速度前进、机器人站着
→ anchor 跑到最大偏差钳位 0.25m 处,`exp(-60×0.25²) = exp(-3.75) ≈ 0.024 ≈ 0`。
即**动的约 64% env 拿不到前进梯度、已放弃该项,真正拿分的只有站立约 36% env**
(正好解释 end=0.43 的构成)。该前进的 env 根本没有梯度指引去追 anchor。

**排除项:** 命令采样健康 —— `stand_still_prob=0.15`、`turn_in_place_prob=0.25`,
约 64% env 拿真实 vx 命令(±0.25,`backward_prob=0.5` 前后对称),命令课程在 walk
已关(`enable_command_curriculum=False`)= 全域从头训,前后左右均在分布内。纯粹是
奖励地形 + 探索问题,非命令分布问题。

## 2. 改法(三项,最小变量,已落地)

以地形为主、探索为辅:

| 项 | 文件:行 | 改动 | 理由 |
|---|---|---|---|
| `rew_k_torso_pos_xy` | `oceanisaaclab_walk_env_cfg.py:95` | 60 → **30** | 0.25m 钳位处仍留 `exp(-30×0.0625)≈0.15` 实梯度,给动的 env 追 anchor 的理由 |
| `rew_w_survival` | `oceanisaaclab_walk_env_cfg.py:114` | 5 → **2** | 降到 5 后仍是前进项 11 倍、依旧主导,继续削站立盆地 |
| `entropy_coef` | `agents/rsl_rl_ppo_cfg.py:74` | 0.0 → **0.005** | 止住 std 崩塌(近因 A),恢复逃出盆地能力;值取小防重新引入不稳定 |

奖励结构变化,**旧 checkpoint 全部作废,须从头重训**。

## 3. 重训与验收

```bash
./_isaaclab/isaaclab.sh -p scripts/rsl_rl/train.py --task Ocean-BDX-Walk-Direct-v0 --num_envs 8192 --headless
```

约 2000~3000 iter 用 `scripts/analyze_tb.py <run_dir>` 交叉验证是否抜出盆地,重点看:

- `Policy/mean_std` 不再塌到 0.01 附近(应稳在 0.05~0.15,说明仍在探索);
- `torso_pos_xy` 明显上升(> 0.6,说明动的 env 开始追 anchor);
- `fall` 不长期贴 0 且 `mean_episode_length` 不骤降(说明在迈步而非纯站)。

## 4. 仍不迈步的下一档杠杆(本轮未动)

- 缩小 `path_frame` 最大位置偏差钳位(0.25m → 更小,让 anchor 始终可达、保住梯度)。

## 5. 第二轮:第一轮探索保住但盆地未破 → 上下一档杠杆

第一轮改动训到 7000 iter(`logs/rsl_rl/bdx_walk_imitation/2026-07-07_11-22-29`):

- ✅ **探索崩塌治住**:`Policy/mean_std` 触底 ~0.035(iter 3000~4000)后反弹稳在
  **0.044**,`Loss/entropy` 同步回升,未再塌到上版 0.013。entropy=0.005 生效。
- ❌ **站立盆地仍未破**:走路指标全部卡在盆地值 —— `torso_pos_xy` 平在 **0.51**
  (未爬过 0.6)、`contact_match` 死平 **1.20**(双脚常着地,非交替)、`leg_joint_pos`
  一度升到 -0.55(iter5000)又回落到 -0.68;`Loss/value` 末段 0.005→0.015 漂移。
  站着能拿的正分(lin_vel 0.84 + contact 1.20 + 站立 env 的 torso_pos)仍压过迈步风险。
  anchor 在 0.25m 钳位处 k=30 也能拿 exp≈0.15,策略宁愿吃低分不追。盆地 7000 iter 已稳,
  再等到 20k 大概率抜不出。
