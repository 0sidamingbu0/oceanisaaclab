## 决策(与用户确认)
- 头部命令 **4-DOF**:(Δh_head 头高, pitch 点头, yaw 摇头, roll 歪头),用满脖子 4 关节。

## 关键设计:脚 IK 与首 IK 解耦
base 固定时脖子在 base 之上、不影响脚 IK。故头部命令**不做成步态库网格维度**(会爆炸),
单独生成一张 4-DOF 头命令→脖子角映射表,运行时四线性插值。现有腿部步态库不用重生成。

## 探明的脖子运动学(placo + URDF)
neck_n4_link 是头。4 关节有效轴(base 系):n1/n2 矢状面对=**头高 Δh + pitch 耦合**、
n3=**yaw**、n4=**roll**。yaw/roll 独立、映射完全线性单调;**头高权限弱**(与 pitch 共用
n1/n2),压到 ±2cm + 连续 warm-start(蛇行遍历)保证映射单调。横向头移动不可实现。

# 阶段1 复盘:加脖子后行走步态退化(sim2sim 蹭脚不抬) + 头命令课程修复(方案 B)

阶段1 加脖子从头训到 13000 iter,**sim2sim 不抬脚、贴地蹭着走**。诊断后确认是**训练本身
步态退化**(局部最优),不是 sim2sim 布局 bug——蹭脚是训练里步态质量下降在 sim2sim 的忠实反映。

## 证据(同 iter 13000 对比,权重相同项)

| 奖励项 | 加脖子前@13k | 加脖子前@21k(成功) | 加脖子@13k |
|---|---|---|---|
| torso_orient | 0.86 | 0.88 | **0.28** ⬇️ |
| torso_pos_xy | 0.93 | 0.94 | **0.53** ⬇️ |
| contact_match | 2.49 | 2.53 | **1.82** ⬇️ |
| leg_joint_pos | -0.74 | -0.64 | -0.78(正常) |
| lin_vel_xy | 0.81 | 0.80 | 0.82(正常) |
| leg 动作 std | 0.066 | 0.061 | **0.044** ⬇️ |

- 腿关节模仿(-0.78)/速度跟踪(0.82)正常 → 表面在走;但躯干朝向(0.28 vs 0.86)、path 位置(0.53)、
  接触模式(1.82)全线塌方。躯干朝向 0.28 = 策略放弃了参考步态的前倾/摆动姿态 → 脚不抬贴地蹭。
- 腿动作 std 从 0.066 塌到 0.044(脖子 4 维 std 更塌到 ~0.006),探索比加脖子前死得更早;且这些
  项从 iter ~8600 就平到 13000——**卡死在更差局部最优,再多训无用**。
- 成功基线 run `logs/rsl_rl/bdx_walk_imitation/2026-07-07_16-14-46`(model_21099),退化 run
  `2026-07-08_17-29-38`。

## 已排除

## 机制(推断)

头命令每 episode 采样后**整段恒定**(yaw±1.0/pitch±0.5/roll±0.6/dh±2cm)。歪头/扭头 =
base 顶上一块**非对称质量偏置**,推离对称参考步态的朝向目标 + 反作用力矩;策略扛不住就降低
探索、放弃精确姿态,退回一个可存活的蹭步。脖子 action_rate/acc -5 也加了"动的代价"。

## 修复:方案 B(头命令课程 + 提探索,保论文范式,脖子仍进 RL)

1. **`oceanisaaclab_walk_env_cfg.py`**:新增 `head_command_curriculum_steps = 120_000`
   (= 5000 iter × 24 steps/env,与扰动课程同 common_step_counter 谱系)。
2. **`oceanisaaclab_walk_env.py`** `_reset_idx`:头命令采样乘
   `hscale = min(1, common_step_counter / head_command_curriculum_steps)`。范围对称于 0,
   `hscale=0` 时头命令全 0、脖子停默认位 → 腿步态先在无头部扰动下长熟,再线性放开到满范围。
3. **`agents/rsl_rl_ppo_cfg.py`** `WalkPPORunnerCfg`:`entropy_coef` 0.005→0.008 作保险,
   延长关键步态涌现期探索(`StandPPORunnerCfg` 自己 override 为 0,不受影响)。
