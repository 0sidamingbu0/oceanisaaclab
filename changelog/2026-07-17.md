# 2026-07-17

## 站立慢速脚距漂移修正

- 对当前站立 ONNX 做 30s 零命令、无外力、200Hz sim2sim 诊断：脚距误差由 `0.54cm`
  增长到 `9.85cm`，双脚始终接触地面；25Hz 以上关节速度能量低于 1%。确认现象主要是策略
  持续向外推脚造成的慢速接触滑移，不是高频控制振动。
- StandPaper 训练分布由约 25% 标准无扰动、25% 错误脚位、50% Table V 扰动调整为
  50% 标准无扰动、20% 错误脚位、30% Table V 扰动，降低“预先扩大支撑面”在训练中的收益。
- 站立 episode 从继承的 8s 延长到 20s，使 10--30s 时间尺度的慢速脚位漂移能在单个 episode
  内持续进入论文 Table I 的腿姿态、速度、动作平滑和躯干跟踪收益；PPO collection 仍为 24 步。
- 不修改论文 Table I 奖励、Table V 扰动力、域随机化、摩擦、动作低通或恢复诊断逻辑。
- 当前 `model_15500.pt` 接口、奖励和 reset 资产均兼容，建议从该点续训 5000--10000 iteration
  做 A/B；目标为 `quiet_foot_drift_cm < 3cm`、`stance_width_error_cm < 1.5cm`，同时保持
  `fall_rate < 8%` 且恢复成功率继续上升。
- 修复错误脚位幅度课程在 `--resume` 后单独从 30% 重启的问题；它现在与 Table V、head 等课程
  一样包含 checkpoint iteration 对应的 control-step offset，`model_15500.pt` 续训时保持 100%。
